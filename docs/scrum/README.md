# Scrum / Proje Yönetimi Kanıtları

Bootcamp, proje yönetimini üründen ayrı puanlıyor. Her sprint sonunda beklenen
belgeler burada tutulur:

- **Backlog dağıtma mantığı** — story → task ayrımı ve puanlama (aşağıda)
- **Daily Scrum notları** — `sprint-N/daily.md`
- **Sprint Board güncellemeleri** — `sprint-N/board.md` (+ ekran görüntüsü)
- **Ürün durumu** — çalışan özelliğin görüntüsü (kök README ekran görüntüleri)
- **Sprint Review** — `sprint-N/review.md`
- **Sprint Retrospective** — `sprint-N/retrospective.md`

## Backlog dağıtma mantığı

- **Story** = kullanıcıya değer veren yetenek (ör. "kullanıcı çamaşırını en ucuz
  saatte çalıştırmayı öneren bir plan görür"). Kök README'deki **Ürün Özellikleri**
  listesi story seviyesidir.
- **Task** = bir story'i hayata geçiren teknik iş kalemi (ör. `optimize` tool'u,
  tarife tool'u, plan ekranı). Sprint tabloları (kök README → Sprintler) task
  seviyesidir ve her biri **story point (SP)** taşır.
- **Puanlama:** Fibonacci-benzeri (1/2/3/5/8). Belirsizlik + efor birlikte
  düşünülür; kodlama içermeyen ama kritik işler (kontrat kilidi) de puanlanır.
- **Sahiplik:** görevler kişilere değil **ekiplere** (YZ / VB / Ortak) etiketlenir;
  "herkes her şeye dokunur" ilkesiyle çift kişi ortak görevleri paylaşır.
